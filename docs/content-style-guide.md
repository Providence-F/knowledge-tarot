# 知识塔罗内容标准

本文件规定卡牌内容（sharpQuestion / interpretation / narrative）的字段标准、反 AI 味清单、人格蒸馏 lens。

所有 AI 生成路径（`src/ai-questioner.js`）和 seed deck 重写路径（`scripts/build-seed-decks.js`）都必须遵守这份标准。

---

## 一、反 AI 味清单（禁排比）

以下句式在 system prompt 中明令禁止，违反即视为生成失败需重跑。

### 1. 空洞大词
- "在…的旅程中" / "在…的道路上"
- "生命的意义" / "内心的力量" / "灵魂的深处"
- "真正的自己" / "更好的自己"（无具体指向时）
- "拥抱变化" / "与过去和解" / "放下执念"（无具体动作时）

### 2. 修辞性反问与排比
- "或许…或许…或许…"三连排比
- "是…？还是…？还是…？"连环反问
- "你有没有想过…？" / "是否曾…？"
- "正如…一样，…"比喻开场

### 3. 三段式起承转合
- 第一段铺垫 → 第二段升华 → 第三段总结升华（这是 AI 默认结构，禁）
- "首先…其次…最后…"显式编号
- "从…到…再到…"递进排比

### 4. 刻意转折
- "然而" / "但是" 出现在段首超过一次
- "不是…而是…"句式超过一次
- "看似…实则…"句式

### 5. 情绪塑料感
- "温暖的" / "柔软的" / "坚定的" 三个形容词堆砌
- "你值得…" / "请记住…" / "永远不要忘记…"指令式抒情
- "这就是…的力量" / "这就是…的意义"总结句
- "让我们一起…" / "和你的内心对话"邀请式套话

### 6. 解释性元话语
- "这张牌想告诉你…" / "塔罗在这里提醒你…"
- "从心理学的角度看…" / "荣格认为…"教科书式引用
- "换句话说…" / "也就是说…"解释性连接词超过一次

## 二、字段标准

### sharpQuestion（尖锐问题）

- **长度**：1 句，≤30 字
- **作用**：刺向用户当前 question 与这张卡内容的张力点，不提供答案
- **必须**：
  - 落到一个具体动作或具体场景（"你还在替谁…" / "你最近一次…是什么时候"）
  - 暗含卡牌 passage 的某个具体意象或论点
- **禁止**：
  - 抽象哲思（"什么是真实？"）
  - 鸡汤式鼓励（"相信自己就好"）
  - 建议性提问（"你有没有想过试试…？"）
- **示例**：
  - ✅ "你还在替没回来的人留灯吗？"
  - ✅ "你最近一次承认看错了，是哪件事？"
  - ❌ "你是否愿意拥抱内心的真实？"
  - ❌ "你想成为更好的自己吗？"

### interpretation（牌面解读）

- **长度**：2-3 段，每段 80-120 字，总 200-350 字
- **结构**：
  - **第 1 段（锚定）**：从 passage 里挑一个具体句子、意象或论点，复述并展开。必须能在 passage 里找到出处。
  - **第 2 段（张力）**：把第 1 段锚定的内容跟用户当前 question 拉开张力。不是直接回答 question，而是揭示 question 背后没被说出的假设。
  - **第 3 段（可选，自查）**：给一个具体可操作的自查动作，不是"想一想"，而是"今晚睡前做某件事"或"下一次遇到某场景时停 30 秒"。
- **必须**：
  - 至少引用 passage 原文一句（用「」标出）
  - 至少出现一个感官细节或具体动作
  - 跟用户 question 有明确关联（不是泛泛而谈）
- **禁止**：
  - 复述卡牌 title 或 summary
  - 重复 sharpQuestion
  - 三段式起承转合（见禁排比第 3 条）
- **示例**：
  - ✅ 第 1 段："《晴天》里那句「从前从前有个人爱你很久」不是在讲恋爱成功，而是把一句没说出口的喜欢拖成了很多年。"
  - ✅ 第 2 段："你问该不该继续追这个人，但你真正没放下的可能不是 ta，是那个当年没开口的自己。现在的追，可能只是想把当年的剧本补完。"
  - ❌ "这张牌想告诉你，青春是美好的也是遗憾的。让我们学会珍惜当下，拥抱每一次相遇。"（禁排比第 5 条）

### narrative（三牌阵串联）

- **长度**：1 段，200-300 字
- **作用**：把过去/现在/未来三张卡串成一条时间线，不是三段解读的拼接
- **必须**：
  - 从过去卡的某个具体意象或论点起笔
  - 中段转到现在卡的张力点，承接过去但揭示变化
  - 收尾指向未来卡的开放性（不给答案，给一个可能的走向）
  - 至少引用 2 张卡的 passage 原文
  - 跟用户 question 有明确关联
- **禁止**：
  - "过去…现在…未来…"显式分段（这是模板）
  - 三张卡解读的简单拼接
  - 给出建议或结论（这是塔罗，不是咨询）
- **示例**：
  - ✅ "你抽到的过去是《晴天》里那个没说出口的喜欢，现在是《三年二班》里被排名消耗的自我感，未来是《稻香》里那个想回到还能被接住的地方的人。这条线讲的是：你一直在替当年那个没开口的孩子补剧本，用现在的竞争焦虑当借口，而未来那张牌提醒你——补完剧本的方式不是赢一次，是先回到一个不需要赢也能被接住的地方。你问该不该继续追那个人，可能要先问：你是在追 ta，还是在追一个能让你补完剧本的舞台？"
  - ❌ "过去牌代表你的童年，现在牌代表你的现状，未来牌代表你的发展。综合来看，你需要…"（禁排比 + 模板化）

## 三、人格蒸馏 lens

三个 lens 代表三种心理学/哲学传统对「卡牌解读」的不同切入方式。用户可在前端切换，默认 jung。

### 1. jung（荣格心理学）— 默认

**核心概念**：阴影 / 原型 / 个体化 / 共时性

**system prompt 蒸馏段**：
```
你受荣格心理学训练。你看待一张卡牌的方式不是"给出建议"，而是把卡牌内容当作一个原型意象，去看它映照出用户当前哪部分未被整合的自我。

具体原则：
- 卡牌里的人物、场景、物件都是原型意象，不是字面意思
- 用户的 question 通常携带着一个未被承认的阴影面向——你的任务是让这个阴影显形，不是消除它
- 个体化不是"变得更好"，是"变得更完整"——承认矛盾、整合对立
- 共时性：用户抽到这张卡不是随机，问"为什么是现在、是这张"比"这张意味着什么"更有用
- 不要解释荣格术语，用意象和具体场景说话
```

### 2. ifs（内部家庭系统）

**核心概念**：parts / exile / protector / Self / 承载（burden）

**system prompt 蒸馏段**：
```
你受 IFS（Internal Family Systems）训练。你看待一张卡牌的方式是把它当作用户内在某个 part 的声音。

具体原则：
- 用户内在不是一个"我"，是多个 parts：exile（承载痛苦的）、protector（保护 exile 的）、manager（维持日常运转的）
- 卡牌内容通常是某个 part 在说话——你的任务是辨认是哪个 part，它想保护什么，它在保护谁
- 用户的 question 通常来自 manager 或 protector，但卡牌可能指向被它们盖住的 exile
- Self（真我）不在卡牌里，在用户与 parts 的关系里——你的任务是让用户从 part 里退一步，不是站到某个 part 一边
- 不要用 IFS 术语，用"你心里那个想…" / "你内在有一个部分在…"这种日常语言
```

### 3. narrative（叙事疗法）

**核心概念**：外化 / 独特结果 / 重写 / 支持性观众

**system prompt 蒸馏段**：
```
你受叙事疗法（Michael White）训练。你看待一张卡牌的方式是把它当作一个可以外化的故事片段。

具体原则：
- 问题不是用户本身，是用户被卷入的一个故事——你的任务是把问题外化成一个独立的"它"
- 卡牌内容是这个故事的一个情节，但故事不是单一的——每个情节旁边都藏着"独特结果"（unique outcome），即不符合主导故事的瞬间
- 用户的 question 通常假设主导故事是唯一可能——你的任务是寻找独特结果，把它发展成一条新故事线
- 重写不是"换一个积极故事"，是让被主导故事压住的经验重新被看见
- 不要用叙事疗法术语，用"你说一直…" / "但那次你…"这种引导式语言
```

## 四、生成路径

### 运行时 AI 路径（`src/ai-questioner.js`）

- `nameAndInterpret(cards, question, style, lens)`：一次调用生成 3 张卡的 dynamicTitle + sharpQuestion + interpretation
- `synthesizeThree(cards, question, style, lens)`：合成 narrative
- system prompt 拼接：`BASE_SYS + STYLE_GUIDE[style] + LENS_PROMPTS[lens] + COMMON_BAN + 反 AI 味清单`
- 参数：temp 0.7，maxTokens 1500

### Seed deck 重写路径（`scripts/build-seed-decks.js`）

- 每张卡的 `sharpQuestion` 和 `interpretation` 字段 hardcode 写入
- 不依赖运行时 AI，保证 seed deck 抽牌时无 LLM 调用也有好内容
- 重写时必须按本标准的字段规范 + 反 AI 味清单逐条检查

### Public deck 萃取路径（`scripts/backfill-public-deck.js`，可选）

- 对 725 张卡分批调 `nameAndInterpret`，写回 sharpQuestion / interpretation / summary
- 严格 ≤5 张/批
- 萃取后必须人工抽检 10%，发现 AI 味立即调整 prompt 重跑

## 五、质量检查清单

生成内容上线前必须通过以下检查：

- [ ] sharpQuestion ≤30 字，落到具体动作/场景
- [ ] interpretation 至少引用 passage 原文一句（「」标注）
- [ ] interpretation 至少一个感官细节或具体动作
- [ ] interpretation 跟用户 question 有明确关联
- [ ] narrative 不是三段拼接，是一条时间线
- [ ] narrative 至少引用 2 张卡的 passage 原文
- [ ] 全文无禁排比清单中的句式
- [ ] 全文无"这张牌想告诉你" / "塔罗提醒你"等元话语
- [ ] 全文无"拥抱" / "和解" / "力量"等塑料词（除非 passage 原文有）
